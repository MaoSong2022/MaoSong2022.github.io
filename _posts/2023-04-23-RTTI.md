# Introduction
面向对象的三大特征为：
- 继承(inheritance)
- 封装(encapsulation)
- 多态(polymorphism)
其中，多态允许我们定义一个指向基类对象的指针，但是该指针指向的对象是派生类实例：

```c++
class Base{
  virtual ~Base() 
}

class Derived: public Base{
}

int main(){
  Base *ptr = new Derived();
}
```

在多态情形下，一个变量的实际类型在运行时才确定，而RTTI, 即Runtime Type Information，就是C++的一个允许一个对象的类型在运行时才被确定的机制。
C++引入RTTI这一机制的原因是许多库都实现了RTTI，从而导致了库间兼容问题，故而C++在语言层面提供了RTTI。
C++引入RTTI的目的是为了让程序在运行时能够根据基类的指针或者引用来获得该指针或引用所指的对象的实际类型。
事实上，RTTI已经能够通过`typeid`操作符来识别出所有的基本类型的变量所对应的指针。

C++通过以下操作来提供RTTI：
1. `typeid`运算符，该运算符返回其表达式或者类型名的实际类型
2. `dynamic_cast`运算符，该运算符将基类的指针或者引用安全地转换为派生类类型的指针或者引用

# typeid
`typeid`运算符接受一个参数，这个参数是一个类型名，或者一个表达式，其返回类型为`const std::type_info&`:

```c++
typeid(type)
typeid(expression)
```

`std::type_info`是C++标准库中记录一个类型实现相关的细节的类，包括这个类型的名字，比较两个类型等，这个类型是无法拷贝的。下面是`type_info`的接口

```c++
class type_info{
public:
  // no copy constructor
  type_info(const type_info &) = delete; 
  // no copy assignment
  type_info & operator=(const type_info &) = delete; 
  virtual ~type_info();
  bool operator==(const type_info &);
  bool operator!=(const type_info &);
  // chech if the type of this type_info precedes the argument 
  // in the implementation's collation order.
  bool before(const type_info &) const;
  // map values of type type_info to a distribution of index values
  size_t hash_code() const;
  bool char* name() const;
}
```

由于`type_info`类无法通过拷贝和赋值得到，因此我们只能通过`typeid`来获得一个`type_info`类的实例。

注意`typeid`的参数类型必须是一个指向含有`virtual`函数的基类的指针解引用(`*p`)，这样`typeid`在运行时确定参数的真正类型，否则该类型在编译的时候就会被确定。
这里需要注意以下几点：
1. 参数必须是一个指针的解引用(`*p`)而不是指针(`p`)本身，否则返回的是指针的类型，而不是指针所指向对象的类型
2. 该指针必须指向一个多态类型，即该类型必须含有`virtual`函数， 否则该参数的类型在编译时就会确定。
我们用如下例子来说明：

```c++
// main.cpp
#include <iostream>
#include <typeinfo>

class Base {
 public:
  virtual void func() {}
};

class Derived : public Base {};

using namespace std;
int main() {
  Derived* pd = new Derived;
  Base* pb = pd;
  cout << typeid(pb).name() << endl;  
  cout << typeid(*pb).name() << endl; 
  cout << typeid(pd).name() << endl; 
  cout << typeid(*pd).name() << endl; 
  delete pd;
}
```

结果输出

```
P4Base
7Derived
P7Derived
7Derived
```

这里`P`表示指针，`4`和`7`表示类型名的长度，最后接这个类型的实际名字。
由于`typeid().name()`的输出与编译器有关，为了得到比较有意义的输出，我们需要用以下命令来编译我们的C++文件：

```
clang++ main.cpp --std=c++11 && ./a.out | c++filt --types
```

这里使用[`c++filt`](https://sourceware.org/binutils/docs-2.16/binutils/c_002b_002bfilt.html)来"demangle"前面输出的[mangling name](https://en.wikipedia.org/wiki/Name_mangling). 原来的输出变成了

```
Base*
Derived
Derived*
Derived
```

由于`pd`和`pb`是一个单纯的指针，因此其类型在编译时就已经确定。
另一方面，`Base`是一个多态类型（含有`virtual`函数），因此`*pd`和`*pb`的类型需要在运行时才能确定，`typeid`会在运行时根据虚函数表来确定其真正类型。我们在后面详细介绍`typeid`确定类型的机制。

除了用来确定参数的真正类型，我们还可以用`typeid`确定template的真实类型：

```c++
#include <iostream>

template <typename T>
T max(T arg1, T arg2) {
  return (arg1 > arg2 ? arg1 : arg2);
}

int main() {
  auto a = max<int>;
  std::cout << typeid(a).name() << std::endl;
  auto b = max<double>;
  std::cout << typeid(b).name() << std::endl;
  return 0;
}
```

结果输出：

```
int (*)(int, int)
double (*)(double, double)
```

# dynamic_cast
`dynamic_cast`运算符被用来将一个表达式`expression`转换为目标类型`type`：

```
dynamic_cast<type*>(expression);
dynamic_cast<type&>(expression);
dynamic_cast<type&&>(expression);
```

这里`type`必须是一个class type并且该类（通常）有`virtual`函数。在第一种情形下，`expression`必须是一个有效的指针；在第二种情形下，`expression`必须是一个左值(l-value);第三种情形下，`expression`必须不是一个左值。
在所有的情形下，`expression`都必须满足以下条件之一：
1. `expression`是`type`所定义类型的派生类
2. `expression`是`type`所定义类型的`public`的基类
3. `expression`和`type`所定义类型一致
如果`expression`的类型是以上三种情形之一，那么`dynamic_cast`调用成功，否则：
1. 如果将`expression`转换为指针类型失败，则结果返回`0`
2. 如果将`expression`转换为一个引用失败，则结果抛出异常`bad_cast`

`dynamic_cast`有两种模式: Upcast和Downcast.
Upcast是将派生类转换为其基类的情形：

```c++
class Base{
  virtual void func(){}
};

class Derived: public Base{}

int main(){
  Derived *ptr_d = new Derived();
  Base *ptr_b = dynamic_cast<Base*>(ptr_d);
}
```

Downcast是将基类转换为派生类的情形：

```
class Base{
  virtual void func(){}
};

class Derived: public Base{}

int main(){
  Base *ptr_b = new Base();
  Base *ptr_d = new Derived();
  Derived *ptr_d1 = dynamic_cast<Derived*>(ptr_b);
  Derived *ptr_d2 = dynamic_cast<Derived*>(ptr_d);
}
```

这里`ptr_d1`比较容易理解，就是简单地将派生类转换为其基类。但是`ptr_d2`不一样，`ptr_d`的类型是`Base *`, 但是其指向的却是一个`Derived`类型的实例。因此，`dynamic_cast`会在运行时根据`ptr_d2`的真正类型来进行转换，即RTTI。
