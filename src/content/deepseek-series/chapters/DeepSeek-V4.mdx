## DeepSeek-V4

DeepSeek-V4 在 DeepSeek-V3 的基础上进一步提高了在长上下文场景下的推理效率。

DeepSeek-V4 包含两个模型，模型配置如下表所示

| Model | DeepSeek-V4 Pro | DeepSeek-V4 Flash |
| --- | --- |---|
| total parameters | 1.6T | 284B |
| activated parameters | 49B | 13B |

DeepSeek-V4 架构上主要有以下创新：

1. 针对 DeepSeek-MoE 架构做了一些小的改动
2. 将 residual connection 改进为 manifold hyper connection (mHC [@mHC])
3. 使用了 HCA, CSA 注意力机制来进一步压缩 KV cache 以及提高 attention计算效率

#### MoE

对于 MoE 模块，DeepSeek-V4 相比于 [DeepSeek-V3](#deepseek-v3) 有以下变化：

<Sidenote>
 see [MoE](https://maosong.website/blog/moe/) for more details.
</Sidenote>

1. 将激活函数从 `sigmoid()` 改进为 `Sqrt(Softplus())`
2. 增加了一个 sequence-wise balance loss 来实现 sequence 层面的负载均衡
3. 移除了node 层面的负载均衡损失，通过并行策略优化来替代这个限制
4. 移除了 DeepSeek-V3 中最开始的 3 层 FFN layers, 改用基于 hash routing 的 MoE layer.

#### mHC

在之前的 LLM 架构中，我们主要使用的是 residual connection [@he2016resnet], 即

$$
X_{\ell+1} = X_{\ell} + f_{\ell}(X_{\ell})
$$

这里 $ x_{\ell}, x_{\ell+1}\in\mathbb{R}^{n\times d}$ 分别是第 $\ell$ 层的输入和输出， $f$ 是 attention layer 或者 FFN layer.

DeepSeek-V4 在这个基础上进行了改进。增强了 residual connection 的表达能力。

之前 Hyper-Connection [@zhu2025hyperconnections] 已经进行了优化，优化后的 residual connection 表达式如下

$$
x_{\ell + 1} = B_{\ell} x_{\ell} + C_{\ell} f_{\ell}(A_{\ell}X_{\ell})
$$

虽然 HC 通过扩展 channel 提高了模型的表征能力，但是，由于矩阵 $B_{\ell}$ 是一个无约束的矩阵，因此很容易带来梯度消失或者爆炸，导致训练不稳定。
为了解决这个问题，DeepSeek-V4 在 HC 的基础上进一步使用了 mHC [@mHC], 其关键思想是 **将 residual mapping 限制到 Birkhoff polytope 上**，其表达式如下：

$$
\begin{aligned}
x_{\ell + 1} &= B_{\ell} x_{\ell} + C_{\ell} f_{\ell}(A_{\ell}X_{\ell})\\
\mathrm{s.t.}\ & B_{\ell}\in\mathcal{M} := \left\{ M\in\mathbb{R}^{m\times m} \mid M\mathbf{1}_m=\mathbf{1}_m, \mathbf{1}_m^TM=\mathbf{1}_m^T, M\geq0 \right\}
\end{aligned}
$$

通过增加 doubly stochastic matrix 这个限制，我们可以保证其谱范数始终小于等与1, 从而保证了残差链接的能力。

#### Attention

DeepSeek-V4 进一步使用了 Compressed Sparse Attention (CSA) 和 Heavily Compressed Attention (HSA) 来提高模型处理长上下文任务的效率。

我们首先介绍 CSA, HCA 相对于 CSA 在两个方面进行了改变

##### Compressed Sparse Attention

Compressed Sparse Attention (CSA) 是 NSA [@NSA] 的一个改进版本。
在 NSA 中，我们构建了三个模块，分别是 token compression, token selection, sliding window 用于挑选上下文的一个子集参与 attention 计算。
CSA 则是先进行 token compression, 然后针对 compressed token 进行 token selection, 再将 selected token 和 sliding window token concate 到一起参与 attention 计算。
并且，为了进一步压缩 KV cache, CSA 使用了 MQA [@MQA].
接下来我们采取 top-down 的形式详细介绍 CSA.

首先，我们使用了 MQA 来计算 attention:

$$
o_{t,i} = \mathrm{MQA}(q_{t,i}, [C_{t}^{slc}; C_{t}^{win}], [C_{t}^{slc}; C_{t}^{win}])
$$

这里 $o_{t,i} $ 是 第 $i$ 个 head 对第 $t$ 个 token 的 attention output.
$K_{t}^{slc},V_{t}^{slc}$ 和 $K_{t}^{win},V_{t}^{win}$ 分别是 token selection, sliding window 对应的 key 和 value.

接下来，我们来介绍 CSA 中的 token selection.
token selection 的目的是挑选出一部分 token 来参与 attention 计算。
之前一般采用 block selection 方法，即将 context 分割为若干个 block, 然后计算每个 block的重要性程度，最后进行挑选。
CSA 在之前的方法上进行了扩展，既然我需要在 block 层面进行计算，为何我不直接存储每个 block 的信息？
也就是说，我现在直接存储 block 对应的 KV cache, 然后再进行挑选与计算。
此时，我们需要两个模块：

1. compressor: 负责将 $m$ 个 token 压缩为 $1$ 个
2. selection: 从压缩后的 token sequence 中进行挑选。

**Compressor**
Compressor 的目的是将一个包含 $m$ 个 token 的 block 压缩为 $1$ 个 entry

其计算逻辑图如下

<HtmlEmbed
  id="deepseek-v4-compressor"
  src="deepseek-v4-compressor.html"
  title="Compute graph of DeepSeek-V4 compressor"
/>

可以看到，在压缩时，压缩后的每个 entry 依赖于其前后两个 block.

计算时，对于第 $i$ 个输出 $C_{i}^{Comp}$ ，其依赖原始输入 hidden states  中的两个相邻的 block.
我们记输入的的 hidden state 为 $H\in\mathbb{R}^{n\times d}$, 这两个相邻的 block 为 $H_{mi:m(i+1)-1}\in\mathbb{R}^{m\times d}$ 和 $H_{m(i-1):mi-1}\in\mathbb{R}^{m\times d}$, 对应的压缩步骤为

$$
C^a = H_{mi:m(i+1)-1}W^{aKV}, C^b = H_{m(i-1):mi-1}W^{bKV}
$$

其中 $W^{aKV}, W^{bKV}\in\mathbb{R}^{d\times c}$, $c<d$ 是 head dimension.
同时，我们还需要像 SwiGLU 的 gate 来计算重要性

$$
Z^a = H_{mi:m(i+1)-1} W^{aZ}, Z^b = H_{m(i-1):mi-1}W^{bZ}, [S^a, S^b] = \mathrm{Softmax}([Z^a + B^a, Z^b + B^b])
$$

这里 $W^{aZ},W^{bZ}\in\mathbb{R}^{d\times c}$, $B^a,B^b\in\mathbb{R}^{m\times c}$.
最后我们基于 weights 和 entry 来得到最终的结果

$$
C_i^{Comp} = \sum_{i=1}^m (S_i^a\odot C_i^a + S_i^b\odot C_i^b)\in\mathbb{R}^c
$$

通过这个方式，我们就将原始的 hidden state $H\in\mathbb{R}^{n\times d}$ 压缩为了 $C^{Comp}\in\mathbb{R}^{n/m\times c}$.

**Selection**
接下来，CSA 进一步通过 selection 的方法提高了 attention 的计算效率。
Selection 主要是通过 Indexer 来实现的。
Indexer 也包括一个 Compressor 用于计算 compressed keys $K^{IComp}\in\mathbb{R}^{n/m\times c^I}$.
接下来就是 index score 的计算

$$
\begin{aligned}
c_t^q &= h_t W^{DQ}\\
q_t &=  [q_{t,1},\dots,q_{t,n_h}] = c_t^QW^{UQ}\\
w_t^I &= [w_{t,1}^I;w_{t,2}^I;\dots,;w_{t,n_h^I}^I] = h_tW^w\\
I_{t,i} &= \sum_{h=1}^{n_h^I} w_{t,h}^I\mathrm{ReLU}(q_{t,h}^IK_{i}^{IComp})
\end{aligned}
$$

最终 CSA 得到的用于参与 MQA 计算的 KV entry 为

$$
C_t^{slc} = \{C_i^{Comp} \mid I_{t,i}\in\mathrm{TopK}(I_{t,:})\}
$$

对于 CSA, 我们需要存储的 KV cache 大小为 $mc+Kc$, 其中 $mc$ 是 sliding window 的 kv cache, $Kc$ 是 选择出来的 top-K entry.

##### Heavily Compressed Attention

与 CSA 不同, HCA 只做 compression 不做 selection, 因此其不包含 Indexer 模块。
并且 HCA 的 compression ratio $m' >> m$ 来保证更高的压缩率。
此时 attention 计算为

$$
o_{t,i} = \mathrm{MQA}(q_{t,i}, C_{t}^{comp}, C_{t}^{comp})
$$

query 与上面计算一致，

$$
c_t^q = h_t W^{DQ}, [q_{t,1},\dots,q_{t,n_h}] = q_t = c_t^QW^{UQ}
$$

compressed entry $C_i^{comp}$ 计算相比 CSA 也更简单，其只需要一个 block $H_{m'i:m'(i+1)-1}\in\mathbb{R}^{m'\times d}$, 计算方式如下

$$
\begin{aligned}
C &= H_{m'i:m'(i+1)-1}W^{KV}\\
Z &= H_{m'i:m'(i+1)-1}W^{Z}\\
S &= \mathrm{Softmax}(Z + B) \\
C_i^{Comp} &= \sum_{i=1}^m (S_i^a\odot C_i^a + S_i^b\odot C_i^b)\in\mathbb{R}^c
\end{aligned}
$$
这里 $W^{KV}, W^Z\in\mathbb{R}^{d\times c}$, $B\in\mathbb{R}^{m'\times c}$.

此时，HCA 的 KV cache 大小为 $\frac{n}{m'}c$, 也就是将序列压缩为了原来的 $1/m'$.

##### Analysis

我们来对比一下不同的 attention （只考虑 single layer）

| Attention | selection | compression | KV cache | Note|
| ---       | ---       | ---         | ---      | --- |
| MHA       | No         | None       | $2n_hd_hL$ |  |
| GQA       | No       |  No         |  $2n_gd_hL$ |  |
| SWA       | latest $K$ | No         | $2n_hd_hK$    |   |
| MLA       |   No        | $d\to c$    |       $(c+d_h^R)L$       | kv sharing  |
| MoBA      | top $K$ blocks    |  No   |  $K* K *2n_hd_hL$  | block size is also $K$ |
| NSA       | top $K$ + latest $K$     |  $L\to L/K$      | $4Kn_hd_h+2n_hd_hL/K$ |
| CSA       | top $K$ + latest $K$   | $d\to c$, $L\to L/ K$ | $(m+K)c$ | kv sharing |
| HCA       | No          | $d\to c$, $L\to L/ K'$ | $\frac{L}{K'}c$  | $K'>> K$, kv sharing |

这里 $L$ 是 sequence length, $n_g, n_h$ 分别是 number of kv heads, number of attention heads, $d_h$ 是 head dimension. 为了方面，这里我们将 sliding window size, topK, compression block size 都设置为 $K$.

可以看到，CSA 和 HCA集成了 NSA 和 MLA 各自的优势，即先基于 MLA 进行降维降低计算量，然后通过 token compression, token selection 来进一步压缩 kv cache.
其中 CSA 使用较小的 compression ratio 来进行细粒度 token selection, 而 HCA 则使用较大的 compression raio 来实现 kv cache 压缩.

##### Other details

1. grouped output projection: 避免 output projection 计算开销太大，计算先降维再升维度。
2. normalization: 使用 QK norm 来提高训练稳定性
3. RoPE: 和 MLA 一样的 partial RoPE.
4. attention sink: 使用和 GPT-oss [@GPT-oss] 一样的技巧来避免 attention sink 的影响

#### Data

训练时，作者首先使用 dense attention 训练 1T token, 然后切换为 sparse attention.

两个导致训练不稳定的问题：

1. Anticipatory Routing: 解耦 routing network 和 backbone network 的更新可以提高训练稳定性。
2. SwiGLU clamping: 使用了 GPT-oss 的 SwiGLU clamping 策略。

### Post-training

Post training 阶段 DeepSeek-V4 将 [DeepSeek-V3.2](#deepseek-v3.2) 的 offline distillation 替换为了 online policy distillation (OPD), 其训练流程如下所示

  title="Post-training of DeepSeek-V4"
/>

每个 specialist 都经过 SFT 和 RL (GRPO) 算法进行训练，训练时，作者为每个 specialist 设计了三种思考模式：

1. Non-think, 直接输出答案
2. Think High, 正常进行思考
3. Think Max, 最强思考模式

对于 RLHF 这类需要 reward model 的任务，作者构建了 rubric-guided RL data 并将模型同时作为 reward model 和 policy model 来进行训练，进而提高模型的表现。

OPD 的损失函数如下所示

$$
\mathcal{L}_{\mathrm{OPD}}(\theta) = \sum_{i=1}^N w_i\mathrm{KL}(\pi_\theta\mid\mid \pi_{E_i})
$$

这里 $E_1,\dots,E_N$ 是对应的 specialist, $\pi_\theta$ 是 policy model, $w_i$ 是对应的权重。

由于现代大模型词表一般较大，因此现有做法一般是使用 single token 来近似 KL divergence, 即

$$
\mathrm{KL}(\pi_\theta\mid\mid \pi_{E_i})\approx \mathrm{sg}\left[\log\frac{\pi_{E_i}(y_t\mid x,y_{<t})}{\pi_{\theta}(y_t\mid x,y_{<t})}\right]
$$

这个估计虽然是无偏的，但是方差很大。
为了解决这个问题，作者使用了原始 KL divergence 定义，对于带来的计算量问题，作者在 infra 层面进行了优化。

对于上下文管理，DeepSeek-V4 没有采用 DeepSeek-V3.2 的做法，而是保留了全部的工具调用内容，让模型能够更好的利用上下文来进行思考。
DeepSeek-V4 针对不同的场景使用了不同的策略：

1. 对于工具调用场景，模型会保留完整的 reasoning history
2. 对于通用对话场景，模型会丢弃之前的思考过程，仅保留上一轮的答案

针对工具调用，作者还介绍了 Quick Instruction, 也就是使用模型本身来进行意图识别。
之前的做法是用一个小模型来进行判断，但是这样无法利用模型的 kv cache, 作者通过在 prompt 之后加入一个特殊的 token 来完成这个任务。
此时模型就可以复用 kv cache 进而降低 latency.

#### Infra

作者在 post-training 阶段使用了 Quantization-Aware Training (QAT, [@QAT]) 来提高模型在 quantization 之后的表现。
作者在两个模块上使用了 FP4  quantization:

1. MoE expert weights
2. QK path

对于 index score, 作者在 QAT 过程中使用了 BF16 精度。

经过训练之后，在推理阶段，作者直接使用 FP4 quantized weights 来进行推理。

对于 OPD, 作者进行了如下优化：

1. 所有 teacher model 的权重都进行 offload
2. 保留 teacher model 最后一层的 hidden states, 避免计算 `lm_head`
3. 按照 teacher model index 来加载数据，使得在一个 mini-batch 里只需要加载一个 teacher model 的 hidden states
4. 使用 TileLang kernel 来优化  `lm_head` 以及 KL divergence的计算

对于长上下文推理，作者使用了 token level WAL, 每生成一个 token 就保留对应的 KV cache, 这样下次重新生成时就可以直接利用已有的 kv cache 继续生成。
