## Engineering

[如何看待由 OpenClaw 作者引发的 "Loop 工程" 讨论？ - 茜布的回答](https://www.zhihu.com/question/2048003050531558553/answer/2049537011460182357) 观点

- Stage 1: prompt engineering (2023-2024), 研究 prompt 怎么写，system prompt 怎么设计，few-shot 怎么组织， CoT 怎么引导
  - prompt 是 AI 时代最重要的技能，但是随着模型越来越强，prompt 的边际效益越来越低
- Stage 2: context engineering (2024-2025), 研究 RAG, Memory, MCP, tool calling, context compression, context window management
  - 真正决定 agent 效果的，不是 prompt, 而是 context
- Stage 3: loop engineering (2026), 研究 goal + loop, 引入持续反馈提高 agent 的稳定性
  - 设计替我写 prompt 的系统，让系统自己递归

作者还将 loop engineering 和 Kubernetes 结合起来
