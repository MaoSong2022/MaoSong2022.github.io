## FSDP

FSDP (fully sharded data parallelism, [@FSDP]) 是 Meta 提出的一个针对大模型训练的数据并行策略。

为了训练大模型，一般我们会采用 pipeline parallelism, tensor parallelisim 或者 Zero-Redundancy parallelism. 但是已有的方法存在两个问题：

1. 一些方法与特定的模型架构绑定，无法迁移到其他模型上
2. 一些方法与特定的框架绑定，一旦框架更新，方法可能就会失效

因此，作者在本文中就提出了一个 robust 以及 efficient 的分布式训练策略，即 **Fully Sharded Data Parallel (FSDP)**, 通过 shard 模型参数的方式来支持大模型的训练。

FSDP 的设计原则有以下几点：

1. User Experience: FSDP 引入了 deferred initialization, 来让用户再不需要 GPU 的情形下构造 model instance
2. Hardware Heterogeneity: FSDP 支持可配置的 sharding 策略，来适配不同的集群
3. Resource Utilization: FSDP 通过 overlap communication 以及 computation 的方式来提高整体的计算效率
4. Memory Planning: FSDP 通过限制 block 的数量来避免内存爆炸

### FSDP2

相比于 FSDP1 将所有的参数展开成一个巨大的 `FlatParameter`,
FSDP2 引入了 `DTensor`, `DTensor` 能够在保持 Tensor 原始结构的前提下，在指定的并行维度上进行更优的切分。
这不仅解决了 FSDP1 中元数据易丢失和 padding 复杂的痛苦，更为 mixed precision training 和 LoRA 提供了支持。

### Why FSDP

1. VLM 架构适配： VLM 的模态交互架构复杂，FSDP 的灵活性比 megatron-lm [@Megatron] 更轻松
2. 敏捷性：对于Qwen3-Next [@Qwen3-Next], gpt-oss [@GPT-oss] 这类新架构，FSDP 可以以更快的速度支持 RL
3. 易用性： FSDP 是一个 pytorch native 的框架，学习曲线与 debug 成本显著低于 megatron-lm
4. 生态兼容：FSDP 能够直接兼容 Huggingface model 格式，不必像 megatron 一样通过 `mbridge` 来进行权重转换
