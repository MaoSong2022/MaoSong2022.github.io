- 不考虑 cpu offload，GRPO 训练时显存里有几个模型，考虑了能省多少显存？

答：GRPO 相比于 PPO, 舍弃了 value model, 因此只有 policy model, reference model, reward model. 但是现在 GRPO 一般使用 verifier 来提供奖励，因此我们这里只考虑policy model, reference model, 其中 policy model 是训练状态, reference model 是推理状态。训练时，一个模型占用的显存包括：参数，梯度，优化器状态，激活值。假设我们使用BF16混合精度训练，前三者的占用分别为 2P, 2P, 12P, 一共 16P, 激活值是一个与 batch size 正相关的量，一般我们不考虑。推理时，一个模型占用的显存就是参数以及 kv cache, 但是这里我们不考虑 kv cache，因为我们只需要 logits, 所以 reference model 的显存占用为 2P. 总哦那个的占用为 18 P. 使用 ZeRO-offload 之后，我们可以将优化器状态 offload 到 CPU 上，这时总的 显存占用为 4P, 降低了 70% 多
