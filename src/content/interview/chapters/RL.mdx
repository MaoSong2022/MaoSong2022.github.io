Q2. 请解释 RLHF 中“奖励模型”与 LLM 作为“策略”之间的关系。用数学符号写出 PPO 在 LLM 场景下的优化目标。

reward model 用于给 LLM 的每一步 (token) 生成对应的奖励.

$$
L = \mathbb{E}_{q\sim D, o_{\leq t}\sim pi_{\theta}(\cdot | mid q)}[\min(r_t(\theta)A_t,\clip(r_t(\theta), 1-\epsilon, 1+\epsilon)A_t)] + \lambda KL(pi_{\theta} \mid\mid \pi_{ref})
$$

Q3. PPO 名字里的“近端”指什么？它与 TRPO 的核心区别是什么？（提示：约束方式）

proximal 代表不要离 reference model 过远， TRPO 通过 KL divergence 来实现约束，而 PPO 通过 clip 来实现，激昂第了复杂度

Q4. 写出 PPO 的 clip 损失函数，并解释 ε=0.2 的含义。如果 Advantage 为正且非常大，clip 会如何影响梯度？

$$
L = \mathbb{E}_{q\sim D, o_{\leq t}\sim pi_{\theta}(\cdot | mid q)}[\min(r_t(\theta)A_t,\clip(r_t(\theta), 1-\epsilon, 1+\epsilon)A_t)] + \lambda KL(pi_{\theta} \mid\mid \pi_{ref})
$$

$\epsilon=0.2$ 代表如果当前 policy 和old policy 之间的概率相差 $20%$, 则说明当前 policy 与 old policy 相差太原，需要进行clip,
Advantage 为正且非常大， 说明当前的动作非常有效，因此梯度会朝着这个方向前进（梯度上升）

Q5. PPO 通常需要价值网络（critic）。在 LLM 场景中，critic 的输入输出是什么？训练 critic 的 label 如何获得？你会将 critic 与 policy 共享底层参数吗？为什么？

critic 负责计算advantage, critic 的输出为当前llm 所能获得的return 的期望，critic 的 label 使用rollout最终的return, 会共享参数，这样value model 训练更方便

Q6. PPO 训练时，如果优势估计（GAE）的 λ 太大或太小，分别会导致什么问题？

lambda 越大，越依赖long term reward information, bias 越小，variance越高，计算复杂度越高，越小月依赖当前的value function estimation, 计算量低，variance低，但是bias 高

Q7. 请说明 GRPO 的核心设计目的——它为什么在数学/代码推理任务中比 PPO 更稳定？GRPO 去掉了 critic，那它用什么来标准化 advantage？

GRPO 通过 group advantage estimation 解决了 PPO 中需要训练 value model的问题，通过 group 内部进行计算，可以更好实现 on policy,

Q8. GRPO 中说“group-based baseline”。假设一个 prompt 生成 8 个回答，reward 分别是 [0.2, 0.5, 0.9, 0.3, 0.4, 0.8, 0.6, 0.1]。请计算每个回答的 advantage（简单归一化方式即可）。
这样做的方差比 PPO 的 GAE 大还是小？为什么还能工作？

Q8-1：这种方法的方差比 PPO-GAE 大还是小？
答案：更大。因为 GAE 利用时序平滑减少了方差，而 GRPO 只用 group 内的瞬时归一化，没有利用时序信息。

Q8-2：那为什么还能工作？

答案：因为数学/代码任务中：每个 token 的重要性相对均匀（不像对话中早期 token 影响巨大） Group 内相对比较已经足够区分好坏去掉了 critic 避免了训练不稳定的问题，trade-off 值得

Q9. PPO 的价值网络会不会被 GRPO 完全替代？在你看来，GRPO 在什么任务上可能不如 PPO？

GRPO 弱于 PPO 的场景：

1. 长程 credit assignment 任务（如长故事生成、多轮对话）
   没有 critic 意味着无法对中间状态进行价值估计，只能靠最终 reward 反推

2. 奖励稀疏（只有一个正确回答，其他全错）
   Group 内如果只有一个正例，advantage 归一化会放大噪声

3. 任务需要细粒度 token-level 优化（如机器翻译、格式约束）
   PPO 的 GAE 能逐 token 调整，GRPO 的 group 归一化是 response-level 的

GRPO 和 PPO 对比

1. 无需value model: 降低显存

### SFT, OPD, RL 对比

结论：SFT 改变模型在外在数据集上的分布，RL改学生在会访问到的state distribution, OPD/OPSD 在 on-policy 状态上使用更dense 的信号

on-policy 仅改变有奖励的局部的区域，其他部分几乎不动，因此 RL 不太可能会遗忘，前提是奖励有意义

RL v.s. OPD: RL 用 advantage 加权， OPD 用 log ratio 加权

| Method | data             | signal                             | risk                                      |
| ------ | ---------------- | ---------------------------------- | ----------------------------------------- |
| SFT    | off policy data  | token level cross entropy          | forgetting                                |
| RL     | on policy states | reward/ verifier                   | rewrighting existed path, low diversity   |
| OPD    | on policy states | teacher logits/priviledged context | mode collapse, over update on style token |

OPD 问题：

1. reversel KL 在 high entropy 上会伤害多样性
2. 强 pass@1 不代表强diversity
