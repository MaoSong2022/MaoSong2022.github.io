agentic RL 相比与 [RLHF](https://maosong.website/blog/RLHF/) / [RLVR](https://maosong.website/blog/RLVR/) 有三个挑战：

1. 决策的自然单位从 token-level 转向了 turn-level, agent 的一次交互包括了 observation, reasoning, tool call, tool calling result 这几个过程，其中的单个 token 不独立面对环境，也不会收到反馈，因此 agentic RL 与 RLHF 和 RLVR 之间存在 granularity mismatch: 后者比前者细两三个数量级
2. Long-horizon tasks: agentic rollout 动辄几十步，上万 token, 很难做 credit assignment, 并且由于 decay 的存在，初始 token 衰减非常严重，而早期的 token 又非常重要
3. sparse outcome reward: agentic tasks 只有任务成功/失败的信号， GRPO [@DeepSeek-Math] 的做法是 rollout 内部所有 token 共享 advantage. 在 agentic 场景下，这会导致数据使用效率低下。

| Method      | RLVR/RLHF    | Agentic RL |
| ----------- | ------------ | ---------- |
| Granularity | token        | turn       |
| Action      | tool calling | token      |
| horizon     | long         | short      |
| State       | POMDP        | MDP        |

用 GRPO 做 agentic RL 的问题： 一条成功 rollout 里也有无效动作，而一条失败 rollout 里也有高质量动作，而 GRPO 则凭结果论英雄，认为成功 rollout 里的动作都是好动作，失败 rollout 里的动作都是坏动作。
其本质上就是将 multi-turn 作为 single-turn 来考虑，并且仅使用 outcome reward 来打分，整个 rollout 成为了一个单一的决策步。

[ LLM Agent RL的一些实践感悟](https://zhuanlan.zhihu.com/p/1979913424113771020)

1. 稳定性是工业级 RL 训练最重要的前提，包括 [Training-Inference mismatch](Training-Inference%20mismatch.md), PPO EWMA 等
2. 对 reasoning RL: algorithm > reward > environment, 对于 agent RL: environment > reward > algorithm
3. 需要稳定进行工具调用的 RL 训练环境
4. 使用 llm-as-a-judge 会出现 reward hacking 问题
5. 减少 hand-craft reward, 避免 reward hacking
6. 对齐训练和评测环境，来保证训练能反映模型的表现
7. environment 和 reward 准备好之后，才有必要进行数据和算法的消融实验
8. 有足够的 GPU 资源是，PPO-EWMA 比 on-policy 效果更好，增大 batch size, group size 和 RL compute
9. RL grokking, on-policy 场景尤其明显
10. 工具层面的探索非常重要，会影响模型能力训练上限
11. 更大模型的 RL 训练泛化效果更好
12. 做 continue RL, 不一定需要 trick, 而是需要高质量数据

为什么要做 credit assignment:

1. reasoning RL: 10k~30K tokens, 比如 deepseek-R1 在 aime2025 上的平均输出上都为 23k tokens. episode-level 方法对每个 token 都使用相同的 advantage
2. agentic RL: 10~100 turns, 总 token 数为 100K~500K, 如 SWE-bench 平均为 131k tokens, 64 turns. episode-level 方法对一个错误的工具调用可能会给和三个相同的正确工具调用相同的奖励

agentic RL 本质上是一个 POMDP 问题

| Paradigm                  | reasoning RL            | agentic RL                       |
| ------------------------- | ----------------------- | -------------------------------- |
| state                     | context $(y_{<t},x)$    | context, environment observation |
| action                    | token $y_t$             | complete response at turn $t$    |
| transition                | concatenation           | stochastic depends on tool use   |
| reward                    | terminal state          | terminal state                   |
| horizon                   | 1 turn, 0.5k-30k tokens | 10-100 turns, 100k-1M tokens     |
| action type               | tokens, homogeneous     | tool, plan heterogeneous         |
| intermediate verification | possible                | rarely possible                  |

agentic RL 的挑战：

1. 环境的变化是随机的
2. POMDP
3. vastly longer horizon
4. heterogeneous action types
5. non-verifiable intermediate states
6. The Bifurcation Point Problem

分类：

1. turn level PRM: agentPRM
2. hindsight and counterfactual methods: reflection
3. cretic-free: GiGPO, POAD
4. hierarchical
5. information theoretic methods: IGPO
6. implicit and DPO based methods
