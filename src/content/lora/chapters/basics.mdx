
## LoRA

### Implementation

```python
import torch
import torch.nn as nn
import math


class LoRALinear(nn.Module):
    def __init__(self, in_features, out_features, r=8, lora_alpha=16):
        self.linear = nn.Linear(in_features, out_features)

        self.linear.weight.requires_grad = False
        if self.linear.bias is not None:
            self.linear.bias.requires_grad = False

        
        self.r = r
        self.lora_alpha = lora_alpha
        self.scaling = lora_alpha / r

        if r > 0:
            self.lora_A = nn.Parameter(torch.zeros(r, in_features))
            self.lora_B = nn.Parameter(torch.zeros(out_features, r))

            nn.init.kaiming_uniform_(self.lora_A, a=math.sqrt(5))
            nn.init_zeros(self.loraB)

    def forward(self, x):
        base_output = self.linear(x)

        if self.r > 0:
            lora_output = (x @ self.lora_A.t()) @ self.lora_B.t()

            return base_output + lora_output * scaling

        return base_output

    def merge_weights(self):
        if self.r > 0:
            delta_w = (self.lora_B @ self.lora_A) * self.scaling

            self.linear.weight.data += delta_w

            self.r = 0
            
```

实际

```python
from peft import LoraConfig, get_peft_model
from transformers import AutoModelForCausalLM

model = AutoModelForCausalLM.from_pretrained("your_model")

config = LoraConfig(
    r=8,
    lora_alpha=16,
    target_models=["q_proj", "v_proj"],
    lora_dropout=0.05,
    bias="none",
    task_type="CAUSAL_LM"
)

model = get_peft_model(model, config)
```

### Advantage

1. mult-tenant serving: 一个 inference engine 可以 serve 多个 adapters
2. layout size for training: LoRA 只需要保存非常小的优化器状态
3. ease of loading and transfer: 只需要保存非常小一部分参数

结论

1. 对于中小规模的 instruction 和 reasoning 数据集，LoRA 和 full fine-tuning 的表现差不多
2. 当数据集规模超过 LoRA 的 capacity 之后，FullFT 的效果比 LoRA 好
3. LoRA 有时候对 large batch size 更敏感
4. 对于比较小的数据，LoRA比全量微调更好，特别是 MLP 和 MoE, attention-only LoRA 比 FullFT 效果差
5. LoRA 在 RL 中 和 FullFT 效果差不多， RL需要非常小的 capacity
