## Recent Progress

### SWA

Qwen2.5-VL [@Qwen2.5-VL] 使用了 sliding window attention (SWA) 来减少 ViT 处理长图片/视频输入时 attention 计算开销。

### ViT-5

模型架构：

1. QK norm, RoPE
2. pre-norm, RMSNorm
3. layer-scale residual

模型细节

- Normalization: pre-norm 使用了 RMSNorm, post-norm 使用了 layer-scale $\mathbf{x}_{l+1} = \mathbf{x}_l+\mathcal{F}(\mathbf{x}_l)\odot \lambda$, layerscale 与 post-norm 表现差不多
- MLP: MLP 使用了 GeLU, 效果比 SwiGLU 好
- position encoding: 联合了 APE 和 2D-RoPE, 与 [CoMP](CoMP.md) 一致
- register token: 使用了 4 个 resiter token
- attention: 使用了 [QK-norm](QK-norm.md), 没有 QKV-bias

### ViT-22B

https://proceedings.mlr.press/v202/dehghani23a.html

#### Parallel Layers

ViT-22B 同时处理 MLP 和 attention, 也就是

$$
\begin{aligned}
y' &=  \mathrm{LayerNorm}(x)\\
y &= x + \mathrm{MLP}(y') + \mathrm{Attention}(y')
\end{aligned}
$$

通过这种方式，我们可以同时处理 $\mathrm{MLP}$ 和 $\mathrm{Attention}$ 的 linear layer. 这样我们可以把不同模块的矩阵乘法进行打包计算，从而提高计算效率。这个方法 [[PaLM]] 页使用了，作者发现，该方法可以在不损失精度的情况下，将计算效率提高 $15\%$.

#### QK Normalization

作者发现，ViT 训练崩溃的原因主要是 attention logits 会爆炸，这就导致了 attention weights 非常接近 one-shot 向量。为了解决这个问题，作者在计算 query 和 key 内积之前，先对 query 和 key 进行一次 normalization, 具体来将，attention score 的计算方式如下：

$$
\mathrm{softmax}\left[\mathrm{LN(XW_Q)}\mathrm{LN(XW_K)}^T/\sqrt{d}\right]
$$

其中, $d$ 是 ViT 的 hidden size, $W_Q, W_K$ 分别是 query projection layer, key projection layer 的权重矩阵。$\mathrm{LN}$ 代表 layer normalization.

#### Omitting Biases

作者还移除了 QKV projection layer 中的 bias, 移除之后，作者发现模型表现没有收到影响，而训练效率提升了 $3\%$ 左右。

对于 MLP 的 bias, 作者进行了保留，作者发现尽管表现有稍微提升 ($0.4\%$), 但是效率上并没有改变。

ViT-22B 的 patch size 为 14, 输入图片精度为 224, 作者使用了和 ViT 一样的可学习位置编码，使用高精度图片进行 fine-tune 的时候，作者使用了 2D interpolation 来进行插值。

模型的配置如下表

| Name    | hidden size | layers | MLP hidden size | Heads | Params(M) |
| ------- | ----------- | ------ | --------------- | ----- | --------- |
| ViT-G   | 1664        | 48     | 8192            | 16    | 1843      |
| ViT-e   | 1792        | 56     | 15360           | 16    | 3926      |
| ViT-22B | 6144        | 48     | 24576           | 48    | 22165     |

训练时，作者使用了 JFT 数据集，包含了 4B 的图片，作者使用 sigmoid cross-entropy loss 进行训练。

### CogViT

GLM-5V-turbo [@GLM-5V-Turbo] 提出了 CogViT, 一个基于 ViT [@ViT] 架构，403M 的 vision encoder.
CogViT 在架构上使用了 QK-Norm [@QK-Norm] 来提高 attention 计算的稳定性。在训练上，作者将训练分为两个阶段：

1. 使用基于蒸馏的 masked image modeling 来提高模型的视觉表征能力，这个阶段使用了 SigLIP2 [@SigLiP2] 和 DINOv3 [@DINOv3] 作为教师模型分别提供 sementic feature 和 textual feature. 训练时图片大小为 $224\times 224$, masking ratio 为 $35\%$. 数据配比为：natural images ($80\%$), instruction following ($10\%$), scientific imagery ($10\%$).
2. 使用对比学习来对齐视觉特征和文本特征。这个阶段使用了 NaFlex 来处理动态分辨率图片输入 以及使用了 SigLIP [@SigLIP] 的 Sigmoid loss (batch size) 来进行训练，数据上，作者加入了 8B 的 bilingual 数据来提高模型的多语种能力。

对于优化器，作者使用了 Muon [@jordan2024muon] 以及 cosine learning rate schedule.
