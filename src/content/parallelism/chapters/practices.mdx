## Practices

一般结合 TP, PP, DP, 设置规则：intra node 设置 TP, 然后 inter node 设置 PP, 最后设置 DP, $\mathrm{DP}=\mathrm{num\_GPU}/(\mathrm{TP}\times \mathrm{PP})$

- 模型显存变为原来的 $1/(\mathrm{TP}\times \mathrm{PP})$
- 梯度显存变为原来的 $1/\mathrm{PP}$

TP 可以降低模型参数和 activation 的内存占用，使用 TP 之后，我们有

$$
\mathrm{Memory} =  \frac{\mathrm{Model}}{\mathrm{TP}} + \mathrm{Gradients} + \mathrm{Optimizer\_States} + \frac{\mathrm{Activation}}{\mathrm{TP}}
$$

PP 只改变模型参数和梯度，因此使用 PP 之后， 我们有

$$
\mathrm{Memory} =  \frac{\mathrm{Model}}{\mathrm{PP}} + \frac{\mathrm{Gradients}}{\mathrm{PP}} + \mathrm{Optimizer\_States} + \mathrm{Activation}
$$

TP, PP, DP (Zero 2) 结合之后，我们按照 TP, PP, DP 的顺序考虑，有

$$
\mathrm{Memory}_{3D} =  \frac{\mathrm{Model}}{\mathrm{PP}\times \mathrm{TP}} + \frac{\mathrm{Gradients}}{\mathrm{PP}\times \mathrm{DP}} + \frac{\mathrm{Optimizer\_States}}{\mathrm{DP}} + \frac{\mathrm{Activation}}{\mathrm{TP}}
$$

### EP

传统的 EP 基于 All to All Dispatch 将 token 分发给不同 rank 上的专家

DeepSpeed 规定了每个专家最多可以计算的 token 数量，将其记为 $p$, 在传输之前，每个 rank 对处理好的 token 进行处理，整理传输给其他显卡的 $(K, p)$ 个 token, 这里 $K$ 是 topK 专家。
这样，我们最终需要传输的数据大小就是 $(K, \mathrm{EP}, p, d)$, $d$ 是 hidden size, 这里我们将当前 rank 也纳入考虑。

接下来，我们对于 rank $i$, 我们保留 $(i, k, p)$ 的数据，然后将 $(\neq i, k, p)$ 的数据发送到对应的 EP rank 上。

传输完之后，每个 rank 上的数据大小还是$(K, \mathrm{EP}, p, d)$, 其中 $\mathrm{EP}-1$ 份数据来自其他 EP rank.

计算时，我们直接进行计算，最后再原路发回即可。
