## Introduction

我们在本节中探讨如何分析 LLM 在训练和推理阶段所需要的显存。

为了回答以上问题，我们先介绍训练/推理阶段的内存计算，再针对可优化部分进行分析并介绍相应优化算法。

### Notation

与参数量、FLOPs 计算所用记号一致；参数量 $P$ 的推导见 [LLM parameter analysis](https://maosong.website/blog/llm-flops-computation/).

|      变量       |         代码          |           含义            |
| :-------------: | :-------------------: | :-----------------------: |
|       $P$       |   `num_parameters`    |   number of parameters    |
|       $L$       |     `num_layers`      |          layers           |
|       $V$       |     `vocab_size`      |      vocabulary size      |
|       $d$       |     `hidden_dim`      |        hidden size        |
| $d_{\text{ff}}$ |  `intermediate_size`  |      FFN hidden size      |
|       $s$       |       `seq_len`       |      sequence length      |
|       $b$       |     `batch_size`      |        batch size         |
|       $h$       | `num_attention_heads` | number of attention heads |
|      $d_h$      |      `head_dim`       | attention head dimension  |

### Assumptions

1. 若无特别说明，使用 **BF16/FP16**，每个参数 **2** byte.
2. 不使用 dropout（与现代大模型设定一致，与 [@korthikanti2022reducingactivationrecomputationlarge] 不同）
3. Attention 基于原始 multi-head attention [@vaswani2017attention]，方便计算
4. FFN 基于 SwiGLU [@SwiGLU]
