RTTI

Posted Apr 23, 2023 Updated May 10, 2023
By Mao Song
9 min read

Introduction

面向对象的三大特征为：

  • 继承(inheritance)
  • 封装(encapsulation)
  • 多态(polymorphism) 其中，多态允许我们定义一个指向基类对象的指针，但是该指针指向的对象是派生类实例：

class Base{
  virtual ~Base() 
}

class Derived: public Base{
}

int main(){
  Base *ptr = new Derived();
}

在多态情形下，一个变量的实际类型在运行时才确定，而RTTI, 即Runtime Type Information，就是C++的一个允许一个对象的类型在运行时才被确定的机制。 C++引入RTTI这一机制的原因是许多库都实现了RTTI，从而导致了库间兼容问题，故而C++在语言层面提供了RTTI。 C++引入RTTI的目的是为了让程序在运行时能够根据基类的指针或者引用来获得该指针或引用所指的对象的实际类型。 事实上，RTTI已经能够通过typeid操作符来识别出所有的基本类型的变量所对应的指针。

C++通过以下操作来提供RTTI：

  1. typeid运算符，该运算符返回其表达式或者类型名的实际类型
  2. dynamic_cast运算符，该运算符将基类的指针或者引用安全地转换为派生类类型的指针或者引用

typeid

typeid运算符接受一个参数，这个参数是一个类型名，或者一个表达式，其返回类型为const std::type_info&:

1
2

typeid(type)
typeid(expression)

std::type_info是C++标准库中记录一个类型实现相关的细节的类，包括这个类型的名字，比较两个类型等，这个类型是无法拷贝的。下面是type_info的接口

class type_info{
public:
  // no copy constructor
  type_info(const type_info &) = delete; 
  // no copy assignment
  type_info & operator=(const type_info &) = delete; 
  virtual ~type_info();
  bool operator==(const type_info &);
  bool operator!=(const type_info &);
  // chech if the type of this type_info precedes the argument 
  // in the implementation's collation order.
  bool before(const type_info &) const;
  // map values of type type_info to a distribution of index values
  size_t hash_code() const;
  bool char* name() const;
}

由于type_info类无法通过拷贝和赋值得到，因此我们只能通过typeid来获得一个type_info类的实例。

注意typeid的参数类型必须是一个指向含有virtual函数的基类的指针解引用(*p)，这样typeid在运行时确定参数的真正类型，否则该类型在编译的时候就会被确定。 这里需要注意以下几点：

  1. 参数必须是一个指针的解引用(*p)而不是指针(p)本身，否则返回的是指针的类型，而不是指针所指向对象的类型
  2. 该指针必须指向一个多态类型，即该类型必须含有virtual函数， 否则该参数的类型在编译时就会确定。 我们用如下例子来说明：

// main.cpp
#include <iostream>
#include <typeinfo>

class Base {
 public:
  virtual void func() {}
};

class Derived : public Base {};

using namespace std;
int main() {
  Derived* pd = new Derived;
  Base* pb = pd;
  cout << typeid(pb).name() << endl;  
  cout << typeid(*pb).name() << endl; 
  cout << typeid(pd).name() << endl; 
  cout << typeid(*pd).name() << endl; 
  delete pd;
}

P7Derived

dynamic_cast

dynamic_cast运算符被用来将一个表达式expression转换为目标类型type：

1
2
3

dynamic_cast<type*>(expression);
dynamic_cast<type&>(expression);
dynamic_cast<type&&>(expression);

这里type必须是一个class type并且该类（通常）有virtual函数。在第一种情形下，expression必须是一个有效的指针；在第二种情形下，expression必须是一个左值(l-value);第三种情形下，expression必须不是一个左值。 在所有的情形下，expression都必须满足以下条件之一：

  1. expression是type所定义类型的派生类
  2. expression是type所定义类型的public的基类
  3. expression和type所定义类型一致

如果expression的类型是以上三种情形之一，那么dynamic_cast调用成功，否则：

  1. 如果将expression转换为指针类型失败，则结果返回0
  2. 如果将expression转换为一个引用失败，则结果抛出异常bad_cast

dynamic_cast有两种模式: Upcast和Downcast.

Upcast是将派生类转换为其基类的情形：

class Base{
  virtual void func(){}
};

class Derived: public Base{}

int main(){
  Derived *ptr_d = new Derived();
  Base *ptr_b = dynamic_cast<Base*>(ptr_d);
}

Downcast是将基类转换为派生类的情形：

class Base{
  virtual void func(){}
};

class Derived: public Base{}

int main(){
  Base *ptr_b = new Base();
  Base *ptr_d = new Derived();
  Derived *ptr_d1 = dynamic_cast<Derived*>(ptr_b);
  Derived *ptr_d2 = dynamic_cast<Derived*>(ptr_d);
}

这里ptr_d1比较容易理解，就是简单地将派生类转换为其基类。但是ptr_d2不一样，ptr_d的类型是Base *, 但是其指向的却是一个Derived类型的实例。因此，dynamic_cast会在运行时根据ptr_d2的真正类型来进行转换，即RTTI。
