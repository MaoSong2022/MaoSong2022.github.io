c++ cast

Posted Sep 6, 2022 Updated Nov 24, 2022
By Mao Song
8 min read

Implicit Conversions

在C++中，在进行不同类型之间类型的运算时，编译器可能会帮助我们将两种类型转化为一种类型再进行操作，这个过程没有任何warning，这个过程就叫做implicit conversion

The Arithmetic Conversions

在算术运算的过程中，隐式转换总是尽可能地保存运算的精度。因此

1

3.541 + 3;

的结果是一个double类型的数，这是因为3.541是一个double， 3是一个int， 而double比int精度高。因此上面的表达式等价于

1

(double)3.541 + (double)3 = (double)6.541

隐式转换结束后，我们会进行赋值或者初始化等行为，因此如果我们进行如下初始化

1

int i = 3.541 + 3;

则i=6, 这是因为我们使用一个double来初始化一个int时，编译器会自动进行一个截断操作，将结果6.541的小数部分都给截断掉。 编译器在以下几种情形会对操作数进行隐式转换：

  • 在大多数表达式中，比int类型小的整数类型会被转换成一个合适的大整数类型。
  • 在条件condition中，非bool类型的表达式会被转换成bool型
  • 在初始化中，初始值会被转换成对应变量的类型；在赋值语句中，等号右边的值的类型会被转换成等号左边变量的类型。
  • 在算术语句和关系语句中，如果表达式含有混合类型的变量的话，它们都将被转换成一个公共类型。
  • 函数调用的过程中，也会出现类型转换。

Other Implicit Conversions

第一个是C语言中的数组和指针之间的转换，即数组名其实是指向第一个元素的指针：

1
2

int ia[10];
int *ip = ia; // convert ia to a pointer to the first element

在指针中，我们可以进行如下转换：

  • 整数0和nullptr可以被转换成任何指针类型
  • 任何指向非const类型的指针都可以被转换成void *
  • 指向任何类型的指针都可以被转换成const void *
  • 指向基类的指针可以被转换成指向派生类的指针（多态）

Explicit Conversions

当我们需要显式转换时，我们就可以用cast来实现这一行为. cast的用法如下：

1

cast-name<type>(expression)

这里cast-name是如下四种之一：

  • static_cast
  • dynamic_cast
  • const_cast
  • reinterpret_cast

static_cast

static_cast主要用于将一个大的数据类型转化为一个比较小的数据类型。static_cast会显式告诉编译器和用户我们意识到并仍然决定要执行操作，尽管这个操作可能会损失精度。

1

int j = static_cast<int>(3.141);

const_cast

const_cast可以修改一个变量的const属性，即cast away the const， 一旦我们去除变量的const属性之后，编译器就不会再阻止我们修改这个变量。const_cast主要用于重载函数(overloaded functions) 只有const_casr才能被用于修改一个表达式的const属性（并且还可能不成功），其他的cast都会导致编译器报错

  1. 如果原始变量没有const修饰的话，我们通过const_cast获得修改权限是合法的。
  2. 如果原始变量是由const修饰的话，那么我们通过const_cast获得修改权限是未定义行为
1
2

const char *pc;
char *p = const_cast<char>(pc); // ok but writing through p is undefined.

reinterpret_cast

reinterpret_cast强制将一个变量改变成目标类型，而不是转换。

1
2

int *ip;
char *pc = reinterpret_cast<char*>(ip);

尽管pc是一个char *类型的变量，但是pc实际上指向的是一个int类型变量，但是编译器并不会报错或者warning，因此使用interpret_cast是非常危险的。

warning: interpret_cast的实现取决于编译器的定义，因此需要非常谨慎地使用。

dynamic_cast

dynamic_cast主要用于将一个基类(base)指针转换成派生类(derived)指针。其主要用法如下：

1
2
3

dynamic_cast<type*>(e);// e must be a valid pointer
dynamic_cast<type&>(e);// e must be a lvalue
dynamic_cast<type&&>(e);// e must not be an lvalue

这里type必须是一个类，并且这个类一般还有虚函数。有以下几种情况需要注意：

  • 如果e是一个指针，则cast失败的话返回nullptr
  • 如果e是一个引用，则cast失败的话抛出一个bad_cast的异常 当我们进行基类Base和派生类Derived之间的转换时，结果有如下几种情况：
    1. type和e的类型相同时，转换总是成功的，返回的就是指向对应类型的指针或者引用
    2. e的类型为Derived， type的类型为Base时，转换也是成功的，因为Derived中包含了Base的数据。
    3. e的类型为Base， type的类型为Derived，此时：
    1. 如果Base中没有虚函数的话，编译器会报错
    2. 如果e指向的对象存储的是Derived实例的话，转换是安全的，正常返回
    3. 如果e指向的对象存储的是Base实例的话，转换不安全，返回nullptr

我们可以使用dynamic_Cast将一个空指针转换为目标类型的空指针

我们可以在if条件中检验dynamic_cast是否成功

1
2
3

 if (Derived *dp = dynamic_cast<Derived*>(bp)){
	 // do something
 }

这样的话，dp的生命周期只在if范围内，可以防止因dynamic_cast出错而出现的错误使用。

总结

C++ primer 上有一个warning：cast本质上是一个危险的构造。 我们需要注意：

  1. 在类型之间进行转换是没问题的
  2. 在指针之间进行转换是需要避免的，除非非常必要
  3. 使用cast之前，应该思考是否存在其他的，能够达到同样效果的操作，以减少或者避免cast的使用。
