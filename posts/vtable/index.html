Virtual Function Table

Posted May 10, 2023 Updated May 29, 2023
By Mao Song
5 min read

这篇博客介绍了多态的实现原理以及虚函数表在多态中的作用。

Introduction

面向对象语言的三大特性：

  • 继承(inheritance)
  • 封装(enclose)
  • 多态(polymorphism)

其中多态的内在机制就是：编译器在运行时根据变量的实际类型采取相对应的行动（调用对应的函数）。而在C++中多态的实现就依赖于虚函数表，这篇博客就记录了学习虚函数表的过程，大部分都是根据之前其他人的笔记整理得到的。

// main.cpp
class Normal
{
public:
 void func1(){};
 void func2(){};
};

class Base {
  virtual void func1(){};
  virtual void func2(){};
};

class Derived: public Base{
  virtual void func1() override{};
};

int main() {
  Normal *normal_ptr = new Normal();
  Base *base_ptr = new Base();
  Base *derived_ptr = new Derived();
  return 0;
}

可以看到，相比于没有virtual函数的Normal， Base类的起始位置多了一个Base vtable pointer， 这就是Base类的指向虚函数表的指针。

Normal的大小不为0的原因是C++不允许有两个不同的对象的地址是一致的，因此一个(空)类的大小(sizeof)至少为1.

Remark

有几点需要注意的是：

  • vptr是在何时被初始化的？由于vptr是一个实例，因此vptr在类的构造函数中被初始化。这也意味着构造函数不能作为虚函数(因为是先调用构造函数，才有虚函数表)
  • vtable是在何时被创建的？这个问题参考文献里通过汇编代码给出了答案：

    编译器在编译器就为每个类确定了vtable的内容，并且在构造函数中添加了相应代码，使得vptr能够指向正确的vtable的地址

  • 调用虚函数的流程？

    首先通过位于栈区的指向类的实例的指针找到位于堆区的实例的地址，然后通过实例内存开头的vptr找到位于只读数据区.rodata里的vtable，再根据vtable里的index找到对应的函数，最后跳转到代码区.text中对应的虚函数代码。这部分详细内容可以参考FAQ.
